Git & GitHub:

Git is a powerful version control system that allows for the user to take digital "snapshots" of previous version of code, allowing for the creation of repositories for version contro, all of this being run in a command line. While learning the ins and outs of Git, you'll learn your basic workflow, which allows for ease of tracking your "snapshots", leading into GitHub. GitHub is a cloud-based Git repository hosting site that allows for multiple users to visually track changes being made to code, as well as collaborate on group projects with other users. The workflow for GitHub is separated into two, one being for initalizing your own repository (initializing git for version control), or "forking"/"cloning" someone elses project. These created repositories live on your command line, as well as on GitHub, so when collaborating on a project, members can "push" the changes they've made to certain things, as well as "pull" the changes made by others.


Some tips for Git and GitHub would be to pay close attention to your teacher. It can be very easy to get lost while learning this, as it will seem very complicated at first. Learning your workflow will also become vital in the future when working on projects within the command line, so make sure to always remember what your workflow is.
